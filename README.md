# asinayo 랜딩 페이지 — 인수인계 문서

브랜드–스타일리스트 협찬 매칭 서비스 **asinayo**의 랜딩 페이지입니다.
이 문서는 배포를 진행할 개발자를 위한 전달 문서입니다.

---

## 1. 구조 한눈에 보기

| 항목 | 내용 |
|---|---|
| 파일 | `index.html` **단 하나** (HTML + CSS + JS 전부 인라인) |
| 프레임워크 | 없음 (Vanilla HTML/CSS/JS) |
| 빌드 도구 | **없음** — npm install / webpack / vite 등 불필요 |
| 백엔드 | **없음** — 100% 정적(프론트엔드) 페이지 |
| 용량 | 약 36KB |

> 별도의 빌드 과정 없이 `index.html`을 정적 호스팅에 올리면 그대로 동작합니다.

---

## 2. 로컬에서 확인하는 법

빌드가 없으므로 파일을 브라우저로 열기만 하면 됩니다.

```bash
# 방법 1) 파일 더블클릭 (가장 간단)
open index.html

# 방법 2) 로컬 서버로 띄우기 (권장)
python3 -m http.server 8000
# → 브라우저에서 http://localhost:8000 접속
```

---

## 3. 배포 방법

정적 파일 하나라 어떤 정적 호스팅에서도 동작합니다. 대표적인 선택지:

| 호스팅 | 방법 |
|---|---|
| **Vercel** | 저장소 연결 또는 폴더 드래그&드롭. 설정 없이 자동 배포 |
| **Netlify** | 폴더를 드래그&드롭하면 즉시 URL 발급 |
| **Cloudflare Pages** | git 연결 후 빌드 명령 비워두고 배포 |
| **GitHub Pages** | 저장소 Settings → Pages에서 브랜치 지정 |
| **AWS S3 + CloudFront** | 버킷에 업로드 후 정적 웹사이트 호스팅 활성화 |

빌드 명령(Build command)을 묻는 경우 **비워두고**, 퍼블리시 디렉터리는 이 폴더(루트)로 지정하면 됩니다.

---

## 4. ⚠️ 실서비스 배포 전 반드시 처리할 항목

현재는 **디자인 시안/샘플** 단계입니다. 그대로 운영에 올리면 안 되며, 아래 항목을 먼저 처리해야 합니다.

### (1) 이미지 — 전부 임시(Unsplash) 사진
- 현재 모든 인물/룩북 이미지는 [Unsplash](https://unsplash.com) 외부 링크로 불러옵니다.
- 외부 핫링크는 **라이선스·로딩 속도·안정성** 문제로 프로덕션에 부적합합니다.
- → 실제 브랜드/스타일리스트 사진으로 교체하고, 이미지를 프로젝트 내부(`/images` 등)에 포함시켜 배포하세요.
- 이미지 URL은 `index.html` 하단 `<script>`의 `P()` / `L()` 헬퍼와 각 데이터 배열(`categories`, `features`, `ring` 등)에 정의되어 있습니다.

### (2) 웨이트리스트 폼 — 동작은 하지만 저장되지 않음
- `#waitlist` 섹션의 이메일 폼은 제출 시 **"신청 완료" 메시지만 표시**하고, 실제로 어디에도 저장되지 않습니다 (프론트엔드 처리만 존재).
- → 실제 수집하려면 백엔드 API 또는 폼 서비스 연동이 필요합니다. 예: Formspree, Google Forms, Supabase, 자체 서버 등.
- 해당 로직: `index.html` 하단 `<script>`의 `waitlistForm` submit 핸들러.

### (3) 텍스트·링크 — 일부 샘플 상태
- 푸터 메뉴 링크, 네비게이션 메뉴는 실제 페이지로 연결되어 있지 않습니다.
- "이미 함께하는 브랜드 — SAMPLE" 로고 스트립은 예시 텍스트입니다.
- → 실제 페이지/링크/브랜드명으로 교체 필요.
